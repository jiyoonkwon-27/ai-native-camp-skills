# Block 0: 맥락의 중요성

> 이 블록은 개념 설명이므로 공식 문서 URL이 없습니다.

## EXPLAIN

**같은 AI인데, 왜 결과가 다를까?**

예시: "다음 주 미팅 안건 만들어줘"

**맥락 없이 물으면:**
> 1. 진행 상황 업데이트 2. 이슈 논의 3. 다음 계획
→ 어떤 팀에나 동일한 뻔한 결과

**6개 도구가 연결된 AI에게 물으면:**
> - [Linear] 스프린트 8 완료율 72%, 블로커 2건
> - [Slack] #product에서 결제 UX 논의 미결
> - [Gmail] 고객사 A SLA 회신 대기 3일째
> - [Calendar] 수요일 디자인 리뷰 → 시안 사전 공유 필요

→ **나만의 상황**에 맞는 구체적 안건이 나옵니다.

**문제**: 이 맥락을 매번 복사-붙여넣기? → 5분이면 지쳐서 결국 안 하게 됨

**해법**:
- **MCP** = 도구를 AI에 직접 연결 (자동으로 맥락 수집)
- **Context Sync 스킬** = "싱크해줘" 한마디로 여러 도구 맥락을 한 문서로 정리

> 오늘 이 두 가지를 배웁니다.

## EXECUTE

**맥락의 차이를 직접 체험해보세요.**

Claude에게 아래 두 가지를 순서대로 요청해보세요:

**1단계: 맥락 없이 요청**
```
다음 주 미팅 안건 만들어줘
```

**2단계: 맥락을 제공하고 요청**
```
나는 PM이고, 우리 팀은 모바일 앱 개발을 하고 있어.
최근 이슈는 결제 UX 개선과 성능 최적화야.
이 맥락으로 다음 주 미팅 안건 만들어줘.
```

→ 두 결과를 비교해보세요. 맥락이 결과를 얼마나 바꾸는지 느껴보세요.
