# Airbridge 딥링크플랜 갱신 안내 업무 자동화 - 문제 정의서

## 문제
Airbridge 딥링크플랜 계약 만료 2달 전에 고객사에 갱신 안내 메일을 보내야 하는데, 대상 파악 + 메일 작성이 반복적

## 스킬 목표
고객사 계약서 PDF에서 계약 정보를 추출하고, 갱신 안내 메일 초안을 Gmail에 자동 생성

## 입출력 정의

### 입력
- **계약서 PDF**: `~/Desktop/업무_계약서/에어브릿지 딥링크 계약서/{고객사명}/` 하위
  - 첫 페이지 = 주문서 (텍스트 추출 불가 → 이미지 변환 후 AI가 읽기)
  - 포함 정보: 고객사명, 담당자 이메일, 플랜명, MAU, 금액, 유료 이용 기간

### 가격 참고
| 사용량 | 연간 금액 |
|--------|----------|
| MAU 5만 | ₩3,200,000 |
| MAU 10만 | ₩4,800,000 |
- 온보딩: 항상 Basic 플랜 (셀프 온보딩)
- 갱신인상률: 7%

### 출력
- **Gmail 초안**: 갱신 안내 메일 (지윤님 검토 후 발송)

## 메일 설정
- 발신자: 권지윤 (jiyoon.kwon@ab180.co)
- 수신자: 고객사 솔루션 도입 담당자 (계약서에서 추출)
- CC: Sales KR Group (sales@ab180.co) — 항상 포함
- 제목: `[{고객사명} | AB180] Airbridge 갱신 안내 (~{만료일})`
- 템플릿: 레드버튼 갱신 안내 메일 기반

## 제약사항
- Salesforce MCP 미지원 → 계약서 PDF에서 직접 정보 추출
- PDF 주문서 텍스트 추출 불가 → 이미지 변환 후 AI 읽기
- Gmail 발송은 초안까지만 → 지윤님 검토 후 수동 발송
