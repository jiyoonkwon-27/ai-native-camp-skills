# 위수탁 점검 양식 자동 기입 스킬

고객사가 보낸 위수탁 점검 양식(xlsx)에 내부 점검 데이터를 매칭하여 자동으로 기입하는 스킬.

## 사용자 입력 받기

스킬 실행 시 AskUserQuestion으로 아래 3가지를 확인:
1. **고객사명** (텍스트 입력)
2. **계약 제품** (airbridge / amplitude / braze 중 택 1)
3. **고객사 양식 파일 경로** (xlsx 파일 경로)

ARGUMENTS가 제공된 경우 거기서 추출하되, 부족한 정보는 질문한다.

## 내부 데이터 경로

```
/Users/ab180-jiyoon-mbp/Downloads/개인정보보호 수탁업체 점검항목(공유용)_2025.09.15의 사본.xlsx
```

이 파일이 **유일한 답변 소스**이다. 고객사 양식 내 기존 답변은 절대 참고하지 않는다.

### 내부 데이터 시트 구조

| 시트명 | 용도 | 참조 조건 |
|--------|------|----------|
| 일반 현황 | 회사 기본 정보 (32개 항목) | 항상 참조. 컬럼: No, 영역, 항목, Airbridge값, Amplitude값, Braze값 |
| Airbridge_2025.09.15 | 에어브릿지 점검 항목 (~100개) | 계약 제품이 airbridge일 때 |
| Amplitude_2025.09.15 | Amplitude 점검 항목 (~100개) | 계약 제품이 amplitude일 때 |
| Braze_2025.09.15 | Braze 점검 항목 (~100개) | 계약 제품이 braze일 때 |

제품별 시트 컬럼: 구분, 점검항목(- 또는 (상세)), 점검항목 텍스트, 근거기준, 평가(Y/N/A), 현황, 증빙자료명

## 실행 흐름

### Step 1: 데이터 로드
Python(openpyxl)으로 두 xlsx 파일을 읽는다:
- 고객사 양식: 시트 구조, 컬럼 헤더, 빈 셀 위치 파악
- 내부 데이터: 일반현황 + 해당 제품 시트 로드

### Step 2: 고객사 양식 동적 분석
고객사마다 양식이 완전히 다르므로 매번 동적으로 파악해야 한다:
1. **어떤 시트에 질문이 있는지** 파악 (여러 시트일 수 있음)
2. **컬럼 역할 식별**: 어떤 컬럼이 질문이고, 어떤 컬럼이 채워야 할 답변인지
3. **비어있는 셀만 대상**: 이미 채워진 셀은 건드리지 않음

### Step 3: 컬럼 매핑
고객사 양식의 컬럼과 내부 데이터 컬럼의 의미를 AI가 판단하여 매핑한다.

일반적인 매핑 패턴:
| 내부 데이터 컬럼 | 고객사 양식 컬럼 (예시, 이름은 다를 수 있음) |
|----------------|------|
| 평가 (Y/N/A) | 준수여부, 평가, 결과 등 |
| 현황 | 준수현황, 답변, 현황, 점검결과 등 |
| 증빙자료명 | 증빙명, 증빙자료, 출처/참고자료 등 |

**처음 보는 컬럼**이 있으면 AskUserQuestion으로 지윤님에게 질문한다.

### Step 4: 질문 매칭
고객사 양식의 각 질문에 대해 내부 데이터에서 가장 잘 맞는 항목을 찾는다.

**매칭 방법**:
1. 고객사 질문 텍스트와 내부 데이터의 점검 항목 텍스트를 비교
2. AI가 의미 기반으로 매칭 판단 (텍스트가 달라도 같은 내용을 묻는지 판단)
3. 일반현황 항목도 함께 매칭 대상에 포함

**매칭 결과 분류**:
- **확신 매칭**: 명확히 같은 내용 → 자동 기입
- **애매한 매칭**: 비슷하지만 확실하지 않음 → 지윤님 확인 필요
- **매칭 실패**: 대응 항목 없음 → 아래 처리

### Step 5: 매칭 실패 처리

| 제품 유형 | 처리 |
|-----------|------|
| Airbridge (자체 제품) | 수동 입력 필요로 표시 |
| Braze/Amplitude (리셀링) | AI가 판단: "AB180이 직접 개인정보를 처리하지 않는 리셀링 맥락에서 이 질문에 답할 수 있는가?" → 답할 수 없으면 "해당없음" 추천, 답할 수 있으면 수동 입력으로 분류. 결과는 지윤님 확인 필수 |

### Step 6: 고객별 가변 항목 식별
내부 데이터에서 아래처럼 플레이스홀더인 항목은 자동 기입하지 않고 "확인 필요"로 표시:
- "계약서 內 수탁 업무명으로 작성"
- "계약서 內 항목으로 작성"
- "고객사 MAU 기준으로 산정하여 작성"
- "담당자(본인) 이름 작성"
- "담당자(본인) 연락처 작성"

### Step 7: 매칭 결과 확인
AskUserQuestion으로 매칭 결과를 보여주고 지윤님의 확인/수정을 받는다.

표시 내용:
- 자동 기입될 항목 개수
- 애매한 매칭 항목 (선택 필요)
- 매칭 실패 항목 (수동 입력 또는 해당없음)
- 확인 필요 항목 (가변 항목)

### Step 8: xlsx 파일 생성
Python(openpyxl)으로 새 xlsx 파일을 생성한다.

**파일명**: `{고객사명} 점검항목 리스트.xlsx`
**저장 위치**: 고객사 양식과 같은 디렉토리

**셀 배경색 규칙**:
- 자동 기입 성공: 배경색 없음 (기본)
- 확인 필요 (가변 항목): **노란색** (FFFF00) 배경
- 수동 입력 필요 (매칭 실패): **주황색** (FFA500) 배경
- 해당없음 (리셀링 AI 판단): 배경색 없음, "해당없음" 텍스트 기입

**중요**: 원본 파일은 절대 수정하지 않는다. 항상 새 파일로 저장.

### Step 9: 증빙파일 zip 생성
매칭 성공한 항목의 증빙자료명을 기반으로 로컬 증빙파일을 찾아 zip으로 묶는다.

**증빙파일 로컬 경로**:
```
/Users/ab180-jiyoon-mbp/Downloads/drive-download-20260323T013911Z-3-001/
```

**폴더 구조** (카테고리별):
| 폴더명 | 대응 카테고리 |
|--------|-------------|
| 01_내부관리계획 | 내부관리계획 |
| 02_개인정보보호 교육 | 내부관리계획 (교육 관련) |
| 03_단말기(PC)보안 & 보안 프로그램 | 보안 프로그램 |
| 04_물리적 접근 방지 & 재해재난 | 물리적 접근 방지, 재해 재난 |
| 05_암호화 | 암호화 |
| 06_접근통제 | 접근통제 |
| 07_접근권한 | 접근권한 |
| 08_접속기록 | 접속기록 |
| 09_출력복사 | 출력/복사 |
| 10_파기 | 파기 |
| 11_개인정보 식별 및 관리 | 개인정보 식별 및 관리 |
| 12. 정보보호 관리체계 | 정보보호 관리체계 |
| 13. 보안 솔루션 운영 | 보안 솔루션 운영 |

**증빙파일 매칭 방법**:
1. 매칭 성공한 각 항목의 `증빙자료명` 컬럼에서 파일명 목록 추출
2. 내부 데이터의 `구분`(카테고리)으로 해당 폴더 결정
3. 해당 폴더 내에서 파일명으로 fuzzy 검색 (증빙자료명과 실제 파일명이 약간 다를 수 있음)
4. 찾은 파일들을 `{고객사명}_증빙파일.zip`으로 압축
5. N/A(해당없음) 항목의 증빙파일은 제외 (증빙자료명이 "-"임)

**zip 저장 위치**: 고객사 양식과 같은 디렉토리

### Step 10: 결과 보고
완료 후 지윤님에게 보고:
- 총 항목 수 / 자동 기입 수 / 확인 필요 수 / 수동 입력 수
- 저장된 파일 경로 (xlsx + zip)
- 증빙파일: 찾은 수 / 못 찾은 수
- 확인 필요 항목이 있으면 "노란색 셀을 확인해 주세요" 안내

### Step 11: 문제 발생 시 슬랙 DM
실행 중 오류가 발생하면 슬랙 DM으로 지윤님에게 알린다.
- slack_send_message 사용
- 오류 내용과 어떤 단계에서 실패했는지 포함

## 중요 규칙

1. **내부 데이터만 답변 소스**: 고객사 양식의 기존 답변은 참고하지 않음 (outdated 위험)
2. **비어있는 셀만 채움**: 이미 채워진 셀은 건드리지 않음
3. **원본 보존**: 항상 새 파일로 저장
4. **확인 필요 항목은 색칠**: 노란색(확인 필요), 주황색(수동 입력)
5. **리셀링 맥락 이해**: Braze/Amplitude는 AB180이 재판매하는 제품으로, AB180이 직접 개인정보를 처리하지 않음. 매칭 실패 시 이 맥락으로 "해당없음" 여부를 판단.
6. **모르면 물어보기**: 처음 보는 컬럼이나 판단이 어려운 경우 AskUserQuestion으로 질문
7. **문제 시 슬랙 DM**: 오류 발생 시 즉시 슬랙으로 알림
