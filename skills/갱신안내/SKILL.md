# Airbridge 딥링크플랜 갱신 안내 메일 자동 생성 스킬

고객사 계약서 PDF에서 계약 정보를 추출하고, 갱신 안내 메일 초안을 Gmail에 생성한다.

## 사용자 입력 받기

스킬 실행 시 AskUserQuestion으로 확인:
1. **고객사 계약서 PDF 경로** — `~/Desktop/업무_계약서/에어브릿지 딥링크 계약서/{고객사명}/` 하위에 있음

ARGUMENTS가 제공된 경우 거기서 추출.

## 제품 정보 (고정)

- **대상**: Airbridge 딥링크플랜만
- **가격**:
  - MAU 5만: 연 ₩3,200,000
  - MAU 10만: 연 ₩4,800,000
- **온보딩**: 항상 Basic 플랜 (셀프 온보딩)
- **갱신인상률**: 7% (계약서 제7조)

## 실행 흐름

### Step 1: 계약서 PDF 읽기
계약서 PDF의 **첫 페이지가 주문서(Order Form)**이다.
텍스트 추출이 안 되므로 **이미지로 변환 후 AI가 읽는다**:

```bash
eval "$(/opt/homebrew/bin/brew shellenv)" && pdftoppm -png -r 200 -f 1 -l 1 "{PDF경로}" /tmp/contract_page
```

변환된 이미지를 Read 도구로 읽어서 아래 정보를 추출:

| 추출 항목 | 주문서 필드명 |
|-----------|-------------|
| 고객사명 | 회사명 |
| 담당자 이메일 | 솔루션 도입 담당자 이메일 |
| 담당자 연락처 | 솔루션 도입 담당자 연락처 |
| 플랜명 | Airbridge Plan |
| 사용량 | 사용량 (MAU) |
| 금액 | 총 이용료 |
| 유료 이용 기간 | 유료 이용 기간 (시작일 ~ 종료일) |
| 만료일 | 유료 이용 기간의 종료일 |

### Step 2: 만료일 확인
추출한 만료일과 오늘 날짜를 비교:
- 만료 2개월 이내 → 갱신 안내 대상
- 만료 2개월 이상 남음 → "아직 안내 시점이 아닙니다" 알림
- 이미 만료 → "이미 만료된 계약입니다" 알림

### Step 3: 메일 초안 생성
Gmail 초안(draft)을 생성한다. **절대 자동 발송하지 않음.**

**메일 제목**:
`[{고객사명} | AB180] Airbridge 갱신 안내 (~{만료일})`

**메일 본문**:
```
안녕하세요, {담당자명}님
AB180 권지윤 입니다.

{고객사명}의 Airbridge 솔루션 갱신 계약 관련 지원을 드리고 있습니다, 잘 부탁드립니다.

다름이 아니라 {고객사명}의 Airbridge 이용 기간이 {만료일}부로 만료를 앞두고 있어,
이용 연장 여부 확인 및 갱신 계약 검토 여부를 확인하고자 연락드렸습니다.

이번 갱신을 통해 보다 안정적으로 서비스를 계속 이용하실 수 있도록 사전에 조건 검토 및 문의를 도와드리고자 합니다.

{고객사명} Airbridge 기존 계약 사항
- 계약 플랜 : {플랜명}
- 계약 사용량: MAU up to {사용량}
- 연간 이용 금액: {금액}
- 계약 만료일 : {만료일}

먼저 솔루션 지속 이용 여부를 내부 논의 후 {회신기한} 내로 이 스레드에 회신해주시면 이어서 이후 절차 안내드리겠습니다.
갱신 계약 관련 추가 문의 사항 있으시면 편히 말씀 부탁드립니다.

감사합니다.
권지윤 드림
```

**회신 기한**: 메일 생성일 기준 약 2주 후 금요일

**메일 설정**:
- To: 고객사 솔루션 도입 담당자 이메일 (주문서에서 추출)
- CC: sales@ab180.co (Sales KR Group) — 항상 포함

### Step 4: 결과 보고
- 추출된 계약 정보 요약
- Gmail 초안 생성 완료 알림
- "Gmail 초안함에서 확인하고 검토 후 발송해 주세요" 안내

### Step 5: 문제 발생 시 슬랙 DM
오류 발생 시 슬랙 DM으로 지윤님에게 알림.

## 계약서 파일 위치

```
~/Desktop/업무_계약서/에어브릿지 딥링크 계약서/{고객사명}/{파일명}.pdf
```

## 중요 규칙

1. **Gmail 초안까지만**: 절대 자동 발송하지 않음. 지윤님 검토 필수.
2. **PDF 첫 페이지 = 주문서**: 텍스트 추출 불가 → 이미지 변환 후 AI가 읽기
3. **메일 톤**: 부드러운 격식체 (~드립니다, ~부탁드립니다)
4. **금액 포맷**: ₩3,200,000 형식 (원화, 천단위 콤마)
5. **모르면 물어보기**: 주문서에서 정보 추출이 불확실하면 지윤님에게 확인
