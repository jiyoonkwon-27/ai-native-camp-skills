# 진행 로그 기록 스킬

현재 대화에서 진행된 작업 내용을 로그 파일에 누적 기록한다.

## 로그 파일 위치

```
~/Documents/claude-logs/{날짜}_{스킬명 또는 주제}.md
```

- 같은 대화에서 여러 번 `/로그`를 실행하면 **같은 파일에 누적** 기록
- 새 대화에서 실행하면 새 파일 생성
- 날짜 형식: YYYY-MM-DD

## 실행 흐름

### 1. 로그 파일 경로 결정
- `~/Documents/claude-logs/` 디렉토리가 없으면 생성
- 파일명: `{오늘 날짜}_{현재 작업 주제를 2-3단어로 요약}.md`
- 같은 대화에서 이전에 로그를 저장한 적 있으면 같은 파일에 이어서 기록

### 2. 현재까지의 진행 상황 요약
대화 컨텍스트를 분석하여 아래 내용을 정리:

```markdown
## {시각} 로그

### 실행한 스킬/작업
- (어떤 스킬을 실행했는지, 무슨 작업을 했는지)

### 주요 결과
- (생성된 파일, 매칭 결과, 의사결정 등)

### 확인/수정 필요 사항
- (사용자가 확인해야 할 것, 미해결 사항)

### 다음 할 일
- (이어서 해야 할 작업)
```

### 3. 파일에 기록
- 기존 내용 아래에 새 로그를 추가 (append)
- 파일 상단에 메타 정보가 없으면 추가:

```markdown
# 작업 로그: {주제}
- 날짜: {날짜}
- 작업자: 권지윤

---
```

### 4. 완료 알림
- 저장된 파일 경로를 알려줌
- 핵심 내용을 1-2줄로 요약하여 보여줌

## 중요 규칙

1. **간결하게**: 로그는 핵심만 기록. 불필요한 설명 제외
2. **누적 기록**: 같은 대화에서는 같은 파일에 이어쓰기
3. **파일 경로 포함**: 생성/수정된 파일이 있으면 반드시 경로 기록
4. **다음 할 일 명시**: 이어서 작업할 수 있도록 다음 단계를 항상 기록
5. **민감 정보 제외**: 고객사 개인정보, API 키 등은 로그에 기록하지 않음
