# 위수탁 점검 업무 자동화 - 문제 정의서

## 문제
위수탁 점검 업무가 주 2회 발생하며, 수동 작업에 시간이 많이 소요됨

## 핵심 병목 2가지
1. **고객사 양식 ↔ 내부 데이터 대조 및 기입** — 고객사마다 양식이 달라서 매칭이 복잡함
2. **증빙파일 수집** — 기입된 증빙파일명으로 구글 드라이브에서 실제 파일을 찾아 모으기

## 입출력 정의

### 입력
- **고객사 양식**: xlsx 파일 (내부에 여러 시트 가능), 파일 경로 직접 입력
  - 고객사마다 양식 구조가 다름 (컬럼 수, 컬럼명, 질문 스타일 모두 상이)
  - 스킬이 매번 양식 구조를 동적으로 파악해야 함 (어떤 컬럼이 비어있고 채워야 하는지)
- **내부 데이터**: 고정된 xlsx 파일 1개 (경로를 스킬에 미리 설정) — **유일한 답변 소스**
  - 파일명: `개인정보보호 수탁업체 점검항목(공유용)_2025.09.15의 사본.xlsx`
  - 고객사 양식 내 기존 답변은 참고하지 않음 (업데이트 안 된 내용일 수 있으므로)
- **계약 제품**: 고객사가 어떤 제품 계약인지 사용자가 함께 알려줌 (airbridge / amplitude / braze)
- **고객사명**: 사용자가 직접 입력 (폴더 생성 및 파일 저장명에 사용)

### 내부 데이터 시트 구조

| 시트명 | 용도 | 컬럼 구조 | 참조 조건 |
|--------|------|----------|----------|
| 일반 현황 | 회사 기본 정보 (32개 항목) | No, 영역, 항목, Airbridge값, Amplitude값, Braze값 | 항상 참조 (제품에 해당하는 컬럼 사용) |
| Airbridge_2025.09.15 | 에어브릿지 점검 항목 (~100개) | 구분, 점검항목, 근거기준, 평가(Y/N/A), 현황, 증빙자료명 | 고객사가 에어브릿지 계약일 때 |
| Amplitude_2025.09.15 | Amplitude 점검 항목 (~100개) | 동일 구조 | 고객사가 Amplitude 계약일 때 |
| Braze_2025.09.15 | Braze 점검 항목 (~100개) | 동일 구조 | 고객사가 Braze 계약일 때 |

### 내부 데이터 핵심 컬럼 (제품별 점검 시트)

| 컬럼 | 설명 | 예시 |
|------|------|------|
| 구분 | 카테고리 | 내부관리계획, 물리적 접근 방지, 암호화, 접근통제, 보안 프로그램 등 |
| 점검 항목 | 질문 텍스트 | "내부관리계획을 수립하고 시행하고 있는가?" |
| 평가 | Y 또는 N/A | N/A = 해당없음 |
| 현황 | 상세 답변 | "ㅇ 개인정보 내부관리계획을 수립하여 운영하고 있으며..." |
| 증빙 자료명 | 증빙파일 목록 | "- 개인정보 내부관리계획\n- 내부관리계획 승인 이력" |

### 리셀링 제품(Amplitude/Braze)의 해당없음 패턴

내부 데이터에 이미 N/A가 명시되어 있음. 별도의 "해당없음 분류" 단계 불필요.

| 카테고리 | Airbridge | Amplitude/Braze |
|----------|-----------|-----------------|
| 내부관리계획 (기본) | Y | Y (공통 사항) |
| 내부관리계획 (접근권한/통제/암호화 상세) | Y | **N/A** (처리시스템 미운영) |
| 물리적 접근 방지 | Y | **N/A** (개인정보 미보관) |
| 암호화 | 대부분 Y | **대부분 N/A** (개인정보 미보관) |
| 접근통제 | Y | **N/A** (처리시스템 미운영) |
| 보안 프로그램 | Y | Y (공통 사항) |
| 재해 재난 | Y | **N/A** (처리시스템 미운영) |

### 출력
- **완성된 양식**: 새 xlsx 파일로 저장 (원본 보존)
  - 파일명: `{고객사명} 점검항목 리스트.xlsx`
  - 자동 기입된 항목: 그대로 기입
  - 고객별 가변 항목 (수탁 업무명, 개인정보 항목 등): **셀 배경색으로 "확인 필요" 표시**
  - 매칭 실패 항목: **셀 배경색으로 "수동 입력 필요" 표시**
- **저장 위치**: 고객사 양식과 같은 디렉토리에 로컬 저장
  - 추후 Google Drive MCP 사용 가능 시 공유 드라이브에 자동 업로드 추가 예정

## 매칭 전략 (fuzzy + AI 조합)

### 매칭 대상
- **일반 현황**: 항목명 매칭 (수탁사명, 회사 주소, CPO, 수탁 업무명, 업무 환경 등)
- **제품별 점검 항목**: 점검 항목(질문) 텍스트 매칭 → 매칭되면 평가/현황/증빙자료명을 함께 기입

### 매칭 흐름
1. **1차: fuzzy 매칭** — 고객사 양식의 질문과 내부 데이터의 점검 항목을 텍스트 유사도로 비교
2. **2차: AI 의미 매칭** — 유사도 70~89% 구간의 애매한 항목만 Claude가 문맥을 파악하여 판단
3. **매칭 성공 시** — 내부 데이터의 평가/현황/증빙자료명을 그대로 기입 (N/A면 "해당없음" 자동 기입)

### 검토 방식 (구간별 자동/수동)

| 유사도 구간 | 처리 방식 |
|------------|----------|
| 90%+ | 자동 기입 (사후 확인) |
| 70~89% | AI가 의미 매칭 후 추천 → 지윤님 선택 |
| 70% 미만 | 매칭 실패 → 아래 "매칭 실패 처리" 참조 |

### 매칭 실패 처리 (제품 유형별 분기)

고객사 양식에는 있지만 내부 데이터에 대응 항목이 없는 경우:

| 제품 유형 | 처리 방식 | 이유 |
|-----------|----------|------|
| **Airbridge** (자체) | 수동 입력 | 자체 제품이므로 대부분 답변이 존재해야 함. 매칭 실패 = 데이터 누락 가능성 |
| **Braze/Amplitude** (리셀링) | Claude가 판단 → 지윤님 확인 | 리셀링이라 진짜 "해당없음"인지, 답변이 필요한데 매칭만 안 된 건지 AI가 문맥으로 구분 |

> **리셀링 매칭 실패 판단 기준**: Claude가 해당 질문의 내용을 읽고,
> "AB180이 직접 개인정보를 처리하지 않는 리셀링 맥락에서 이 질문에 답할 수 있는가?"를 판단.
> - 답할 수 없음 → "해당없음" 추천
> - 답할 수 있음 → 수동 입력으로 분류
> - 결과는 반드시 지윤님이 확인/수정

## 실패 리스크
- 잘못된 정보가 고객사에 전달될 수 있음 (매칭 오류 시)
- 대응 1: 구간별 처리로 애매한 매칭은 반드시 사람이 확인
- 대응 2: 메일 발송 전 최종 검토

## 스킬 흐름 (`/위수탁점검`)

1. 사용자가 고객사 양식 경로 + 계약 제품(airbridge/amplitude/braze) 입력
2. 고객사 양식 시트 구조 동적 파악 (질문 컬럼, 빈 컬럼 식별) & 내부 데이터 로드
3. 컬럼 매핑: AI가 고객사 컬럼 ↔ 내부 데이터 컬럼 의미 매칭 (모르는 컬럼은 지윤님에게 질문)
4. fuzzy + AI 매칭으로 항목 대조 & 추천 (N/A 항목은 자동으로 "해당없음" 기입)
5. 매칭 실패 항목 처리:
   - Airbridge → 수동 입력으로 분류
   - Braze/Amplitude → Claude가 "해당없음 vs 수동 입력" 판단 → 지윤님 확인
6. 고객별 가변 항목 (수탁 업무명 등) → 확인 필요로 색칠 표시
7. 지윤님 전체 매칭 결과 확인/수정
8. 새 xlsx 파일로 양식 자동 기입 (확인 필요 항목은 배경색 표시)
9. 증빙파일 구글 드라이브에서 자동 수집
10. 문제 발생 시 → 슬랙 DM으로 지윤님에게 알림

> **추후 추가 예정**: Gmail 메일 초안 생성 (완성본 + 증빙파일 → 검토 후 발송), Notion CRM 업데이트

## 고객사 양식 특성 (실제 샘플 분석 기반)

고객사마다 양식이 크게 다르며, 유형을 사전에 분류할 수 없음. 스킬이 매번 동적으로 파악해야 함.

### 양식 간 차이점 예시

| 항목 | 카카오페이손해보험 (Airbridge) | 코빗 (Braze) |
|------|------|------|
| 시트 수 | 2개 (점검표 + 증빙자료) | 2개 (체크리스트 + 기존답변) |
| 질문 수 | 37개 | ~32개 |
| 채울 컬럼 | 준수여부(Y/N/NA), 준수현황, 증빙명 | 답변, 출처/참고자료 |
| 질문 스타일 | 법적/제도적 ("내부관리계획 수립 여부") | 실무/기술적 ("IP Whitelist 설정 가능한지") |

### 스킬이 동적으로 파악해야 할 것
1. **어떤 시트에 질문이 있는지** — 시트가 여러 개일 수 있음
2. **어떤 컬럼이 질문이고, 어떤 컬럼이 비어있어 채워야 하는지**
3. **비어있는 칸만 채움** — 기존에 채워진 내용은 무시 (outdated 위험)

## 미해결 사항
- **제품 기능 질문 커버리지 갭**: 코빗 같은 양식에서 Braze/Amplitude 제품 자체의 기능을 묻는 질문은 내부 데이터로 답변 불가. Privacy팀에서 실제 기입한 결과물을 받으면 보완 예정.

## 기타 맥락
- 고객사 양식: 구글 시트로 받아 xlsx로 다운로드하여 사용
- 증빙파일: 구글 드라이브에 보관 (파일명 + 폴더 구조로 검색)
- 팀 공유: 완성본을 팀원에게 전달하는 수준 (도구 자체를 공유할 필요 없음)
- Notion CRM 연동은 추후 세팅 완료 후 추가 예정 (현재 스킬에서 제외)
- 내부 데이터 파일 위치: ~/Downloads/ (향후 스킬 설정에 고정 경로 지정)
